# Separate Deployment Guide: Backend on Railway + Frontend on Netlify

## Overview

This guide explains how to deploy the voice emotion analysis application with:
- **Backend**: Railway (Node.js with OpenAI Whisper + GPT-4)
- **Frontend**: Netlify (Static HTML/CSS/JS)

## Architecture

```
Frontend (Netlify)  -->  Backend API (Railway)  -->  OpenAI API
     |                        |                        |
  Static Site            Node.js Server         Whisper + GPT-4
  voice-abhinaykalal    voice-emotion-api       97% Accuracy
  .netlify.app          .up.railway.app
```

## Backend Deployment (Railway)

### 1. Push Backend Code to GitHub

```bash
# Create new repository for backend
cd backend
git init
git add .
git commit -m "Initial backend with OpenAI integration"
git remote add origin https://github.com/yourusername/voice-emotion-backend.git
git push -u origin main
```

### 2. Deploy to Railway

1. **Go to [railway.app](https://railway.app)**
2. **Click "New Project"**
3. **Connect GitHub repository**
4. **Select `voice-emotion-backend` repository**
5. **Add Environment Variables**:

```bash
# Required
OPENAI_API_KEY=your_openai_api_key_here

# Optional (for database storage)
SUPABASE_URL=your_supabase_project_url
SUPABASE_ANON_KEY=your_supabase_anon_key
USE_SUPABASE=true

# Railway Configuration
NODE_ENV=production
PORT=3000
FRONTEND_URL=https://voice-abhinaykalal.netlify.app
```

6. **Click "Deploy"**
7. **Wait for deployment** (2-3 minutes)
8. **Copy the Railway URL** (e.g., `https://voice-emotion-api-production.up.railway.app`)

### 3. Test Backend

```bash
# Health check
curl https://your-backend-url.up.railway.app/api/health

# Expected response
{
  "status": "ok",
  "message": "Voice Emotion Analysis API is running on Railway",
  "timestamp": "2024-01-01T00:00:00.000Z",
  "version": "2.0.0",
```

## OpenAI API Setup

### 1. Get OpenAI API Key

1. **Go to [OpenAI Platform](https://platform.openai.com)**
2. **Sign up or log in**
3. **Navigate to API Keys**
4. **Click "Create new secret key"**
5. **Copy the key** (starts with `sk-`)

### 2. Configure OpenAI

Add the OpenAI API key to Railway environment variables:

```bash
OPENAI_API_KEY=sk-your-actual-openai-api-key-here
```

## Supabase Setup (Optional)

### 1. Create Supabase Project

1. **Go to [Supabase](https://supabase.com)**
2. **Click "Start your project"**
3. **Create new project**
4. **Get project URL and anon key**

### 2. Create Database Table

```sql
-- Create emotion_analyses table
CREATE TABLE emotion_analyses (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  primary_emotion VARCHAR(50) NOT NULL,
  emotion_data JSONB NOT NULL,
  confidence DECIMAL(3,2),
  analysis TEXT,
  transcription TEXT,
  audio_features JSONB,
  audio_size INTEGER,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Enable RLS
ALTER TABLE emotion_analyses ENABLE ROW LEVEL SECURITY;

-- Create policy
CREATE POLICY "Allow all operations" ON emotion_analyses
  FOR ALL USING (true);
```

### 3. Configure Supabase

Add Supabase environment variables to Railway:

```bash
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_ANON_KEY=your-supabase-anon-key
USE_SUPABASE=true
```
